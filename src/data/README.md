# Portfolio Content

Edit `projects.js` to add, remove, or update portfolio projects.

Each project object creates:
- A homepage work card
- A separate `/work/project-id` case-study page
- The project detail sections, outcome, learning, and gallery

Use a unique `id` such as `new-mobile-app`. The page URL becomes `/work/new-mobile-app`.

For images, use either:
- `image("framerImageId")` for Framer image assets
- A full image URL string, for example `"https://example.com/project.png"`

Keep these fields filled for the best SEO and case-study layout:
- `id`
- `eyebrow`
- `title`
- `client`
- `platform`
- `role`
- `objective`
- `cover`
- `alt`
- `summary`
- `problem`
- `users`
- `approach`
- `challenges`
- `decisions`
- `outcome`
- `learned`
- `gallery`
